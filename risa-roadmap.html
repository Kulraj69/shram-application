Why Dashboard Utilization Stalls at <100%

Root Cause Analysis

Missing Features

Users work around gaps in functionality instead of using the dashboard for certain workflows.

Root Cause: Feature completeness debt accumulated over time

Trust Deficit

PMs double-check dashboard outputs against other sources before making decisions.

Root Cause: Analytics gave wrong values, data reliability issues in the past

Workflow Disconnects

Steps happen outside the dashboard, breaking the operational flow.

Root Cause: Writeback gaps, manual handoffs, external dependencies

Knowledge Gaps

"I don't know if the dashboard can do X" — feature discovery is poor.

Root Cause: No in-app guidance, no feature catalog, no training

Approval Gaps

Approvals require email/Slack instead of in-dashboard workflows.

Root Cause: Missing approval systems and audit trails

Reporting Gaps

Leadership pulls raw data instead of using pre-built executive reports.

Root Cause: No executive dashboards, no scheduled reports

Phase 1: Quick Wins

Week 1-2

Remove the immediate blockers that prevent users from using the dashboard fully. Low effort, high impact. These are the "why isn't this already done" items.

1. Feature Discovery Overlay
Low Effort

Problem: Users don't know what the dashboard can do. They manually query BigQuery or use workarounds because they don't know the feature exists.

Fix: In-app feature tooltips + "New in Dashboard" notification banner + searchable feature help.

+15% adoption among occasional users

2. Status Comment Fetch from CMM
Medium Effort

Problem: Status shows "Sent to Plan" but no plan response details visible. Users must login to CoverMyMeds separately.

Fix: Auto-fetch CMM status comments on status page. Display plan decision notes inline.

GET /api/v1/status/{id}/cmm-comments
Response: {
  "decision": "Approved",
  "effective_date": "2026-05-01",
  "comments": "Prior auth approved for 12 months",
  "pdf_url": "https://..."
}

-100% manual CMM logins for status checks

3. Dev/Prod Environment Toggle
Low Effort

Problem: PMs confused about which dashboard to use. Accidental prod actions during testing.

Fix: Clear environment indicator (color-coded banner) + prominent one-click link to switch.

-100% prod accidents from testing

4. Missing Data Pre-Filter
Low Effort

Problem: Users scan entire list looking for actionable items. Time wasted on items already complete.

Fix: One-click "Show only items with missing data" filter. Prominent toggle in filter bar.

-50% time to find actionable items
